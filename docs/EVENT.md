# EVENT.md

이 문서는 `tao logue`의 이벤트 시스템을 정리하기 위한 기획 문서다.

확정된 이벤트는 추후 JSON 데이터로 옮겨 게임에서 사용한다. 이 문서는 사람이 읽는 설계 문서이고, JSON은 게임이 읽는 실행 데이터로 분리한다.

## 이벤트의 역할

이벤트는 고정 스테이지가 아니라 조건을 만족했을 때 발생하는 변주 요소다.

- 스토리와 세계관을 보여준다.
- 이전 선택의 결과를 플레이어에게 돌려준다.
- 혼돈치, 숨겨진 아이템, 동료, 보스 처리 선택을 게임 진행에 연결한다.
- 전투나 정비만으로 설명하기 어려운 보상과 리스크를 제공한다.

이벤트는 플레이어를 항상 강하게 만드는 장치가 아니다. 좋은 보상, 손실 회피, 정보 획득, 혼돈 변화, 숨겨진 조건 개방 중 하나 이상의 의미를 가져야 한다.

## 이벤트 발생 조건

이벤트는 아래 조건을 조합해 발생시킨다.

- 월
- 난이도
- 혼돈치 구간
- 보유 카드
- 보유 아이템
- 숨겨진 아이템 획득 여부
- 동료 보유 여부
- 이전 이벤트 선택
- 보스 처리 선택: 카드화, 소멸, 봉인
- 전투 결과
- 정비 선택

같은 조건의 이벤트가 너무 자주 반복되지 않도록, 이벤트별 1회성 여부와 재등장 가능 여부를 분리한다.

## 이벤트 종류

초기 이벤트 종류:

- 스토리 이벤트: 세계관, 인물, 월별 분위기를 전달한다.
- 혼돈 이벤트: 혼돈치에 따라 대사, 선택지, 결과가 달라진다.
- 숨겨진 아이템 이벤트: 조건을 만족하면 숨겨진 아이템을 발견하거나 획득한다.
- 동료 이벤트: 동료를 만나거나 잃거나, 동료의 상태가 바뀐다.
- 보스 후속 이벤트: 카드화, 소멸, 봉인 선택의 결과가 이후 월에 나타난다.
- 정비 이벤트: 정비 중 이상 징후나 특별 거래가 발생한다.
- 파국 이벤트: 혼돈치가 100에 도달했을 때 게임을 강제로 끝내는 이벤트다.

## 이벤트 선택지 원칙

이벤트 선택지는 단순한 정답 찾기가 아니라 방향 선택이어야 한다.

- 안전한 선택: 손실이 적지만 보상도 낮다.
- 위험한 선택: 혼돈, 체력, 카드, 아이템을 대가로 큰 보상을 얻는다.
- 정보 선택: 즉시 보상은 낮지만 숨겨진 조건이나 보스 정보를 얻는다.
- 거절 선택: 이벤트를 지나치고 현재 상태를 보존한다.

선택지가 있는 이벤트는 최소한 아래 중 하나를 변화시켜야 한다.

- 혼돈치
- 체력
- 재화
- 카드 추가, 제거, 강화, 봉인
- 아이템 획득 또는 손실
- 동료 상태
- 숨겨진 플래그
- 이후 이벤트 또는 엔딩 조건

## 혼돈치와 이벤트

혼돈치는 이벤트의 발생 조건과 결과를 바꿀 수 있다.

- 안정: 기본 이벤트가 주로 등장한다.
- 흔들림: 일부 이벤트 대사와 전투 전조가 변한다.
- 균열: 숨겨진 아이템 획득 조건에 가벼운 미니게임이나 추가 판정이 붙을 수 있다.
- 침식: 카드 봉인, 카드 삭제 압박, 특정 카드 사용 강제, 특정 카드 지정 같은 변칙이 이벤트 결과로 등장할 수 있다.
- 파국: 강제 이벤트가 발생해 게임이 바로 끝난다.

이벤트 왜곡은 아직 보류 중이다. 적용한다면 같은 이벤트라도 혼돈치에 따라 대사, 선택지, 결과가 바뀌는 방식으로 검토한다.

## 숨겨진 아이템 이벤트

모든 월에는 숨겨진 아이템을 배치한다.

숨겨진 아이템은 단순한 보물상자가 아니라, 아래 조건 중 하나 이상을 요구하는 방식으로 설계한다.

- 특정 월의 전투를 특정 방식으로 클리어
- 특정 카드 타입 보유
- 특정 보스 처리 선택 이력
- 특정 혼돈치 구간
- 특정 정비 선택
- 가벼운 미니게임 성공
- 이전 월에서 얻은 단서 보유

숨겨진 아이템 중 일부는 강력한 대신 혼돈치를 증가시킬 수 있다.

## 파국 이벤트

혼돈치가 100에 도달하면 파국 이벤트가 발생한다.

파국 이벤트는 보스를 다른 존재로 바꾸는 방식보다, 현재 보스와 플레이어의 관계를 강제로 결론짓는 방식으로 설계한다.

후보:

- 보스의 아래로 들어간다.
- 보스와 즉시 결투하고, 패배 또는 승리로 게임이 끝난다.
- 승리 후 플레이어가 새로운 보스가 된다.

새로운 보스가 되는 엔딩은 2회차 이후 개방하는 방향을 검토한다.

## JSON 데이터화 방향

이벤트는 추후 JSON 파일로 옮겨 관리한다.

초기 JSON 필드 후보:

```json
{
  "id": "event_january_hidden_flower_01",
  "title": "숨겨진 꽃패",
  "type": "hidden_item",
  "month": 1,
  "once": true,
  "conditions": {
    "chaos_min": 0,
    "chaos_max": 49,
    "required_flags": [],
    "required_items": [],
    "required_cards": []
  },
  "script": {
    "default": "낡은 장독대 뒤에서 희미한 기운이 느껴진다.",
    "hard": "장독대 뒤의 기운은 처음부터 너를 기다렸다는 듯 흔들린다."
  },
  "choices": [
    {
      "id": "inspect",
      "label": "살펴본다",
      "requirements": {},
      "effects": {
        "gain_item": "hidden_flower_january",
        "set_flags": ["found_january_hidden_item"]
      }
    },
    {
      "id": "leave",
      "label": "그냥 지나친다",
      "requirements": {},
      "effects": {}
    }
  ]
}
```

JSON은 기획 확정 후 `Data/Events/` 같은 별도 폴더에 월별로 나누는 방향을 검토한다.

예상 구조:

```text
Data/
  Events/
    month_01.json
    month_02.json
    chaos_events.json
    boss_aftermath_events.json
```

## 미정 사항

- 이벤트 왜곡을 실제 시스템으로 넣을지 여부
- 이벤트 선택지 수의 기본값
- 이벤트 보상과 전투 보상의 상대 가치
- 숨겨진 아이템 미니게임의 구체 방식
- 동료 이벤트를 초기 버전에 넣을지 여부
- JSON 스키마를 어느 시점에 고정할지
