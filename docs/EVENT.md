# EVENT.md

이 문서는 `tao logue`의 이벤트 시스템을 정리하기 위한 기획 문서다.

확정된 이벤트는 추후 JSON 데이터로 옮겨 게임에서 사용한다. 이 문서는 사람이 읽는 설계 문서이고, JSON은 게임이 읽는 실행 데이터로 분리한다.

## 이벤트의 역할

이벤트는 고정 스테이지가 아니라 조건을 만족했을 때 발생하는 변주 요소다.

- 스토리와 세계관을 보여준다.
- 이전 선택의 결과를 플레이어에게 돌려준다.
- 혼돈치, 숨겨진 아이템, 동료, 보스 처리 선택을 게임 진행에 연결한다.
- 전투나 정비만으로 설명하기 어려운 보상과 리스크를 제공한다.

이벤트는 플레이어를 항상 강하게 만드는 장치가 아니다. 좋은 보상, 손실 회피, 정보 획득, 혼돈 변화, 숨겨진 조건 개방 중 하나 이상의 의미를 가져야 한다.

## 이벤트 발생 조건

이벤트는 아래 조건을 조합해 발생시킨다.

- 월
- 난이도
- 혼돈치 구간
- 보유 카드
- 보유 아이템
- 숨겨진 아이템 획득 여부
- 동료 보유 여부
- 이전 이벤트 선택
- 보스 처리 선택: 카드화, 소멸, 봉인
- 전투 결과
- 정비 선택

같은 조건의 이벤트가 너무 자주 반복되지 않도록, 이벤트별 1회성 여부와 재등장 가능 여부를 분리한다.

## 이벤트 종류

초기 이벤트 종류:

- 스토리 이벤트: 세계관, 인물, 월별 분위기를 전달한다.
- 혼돈 이벤트: 혼돈치에 따라 대사, 선택지, 결과가 달라진다.
- 숨겨진 아이템 이벤트: 조건을 만족하면 숨겨진 아이템을 발견하거나 획득한다.
- 동료 이벤트: 동료를 만나거나 잃거나, 동료의 상태가 바뀐다.
- 보스 후속 이벤트: 카드화, 소멸, 봉인 선택의 결과가 이후 월에 나타난다.
- 정비 이벤트: 정비 중 이상 징후나 특별 거래가 발생한다.
- 파국 이벤트: 혼돈치가 100에 도달했을 때 게임을 강제로 끝내는 이벤트다.

후반부 확정 이벤트 후보:

- 11월 봉황 이벤트: 11월은 엑스트라 덱을 제공하지 않고 봉황과 관련된 확정 이벤트로 처리한다. 봉황은 풍요와 뛰어나고 청렴한 선비를 상징하는 영물로 해석한다.
- 12월 최종 보스 이벤트: 12월은 엑스트라 덱을 제공하지 않고 최종 보스 흐름에 집중한다. 보스는 비광 속 인물이며, 비광 속 인물은 주인공의 그림자다.

11월 봉황 이벤트 흐름:

- 11월 보스는 봉황이다.
- 봉황 정화 후 플레이어와 소통하는 이벤트 장면이 나온다.
- 공통 보상으로 봉황의 깃털을 제공한다.
- 봉황의 깃털은 12월 비광 속 인물 정화에 필요하다.

혼돈치별 봉황 보상:

- 혼돈치 0: 봉황의 선택을 받는다. 봉황이 12월 보스전에서 함께 전투하고, 보스 정화도 직접 돕는다. 가장 좋은 루트다.
- 혼돈치 낮음: 봉황의 깃털을 받고, 12월 보스에 대한 중요한 힌트와 작은 축복 또는 방어적 버프를 받는다.
- 혼돈치 높음: 봉황은 12월 보스에 대한 힌트만 남기고 깃털은 주지 않는다.

후보 대사:

```text
봉황: "당신이 지나온 길을 보니 절대 그를 이길 수 없을 것 같군요. 부디 이기십시오, 나의 썩을 주인이여."
```

봉황 조우 대사 후보:

```text
봉황: "오셨군요, 나의 썩을 주인이여."
주인공: "너는 나를 기억하는구나."
봉황: "..."
```

봉황은 현재 주인이 바뀐 상태지만 주인공을 보고 주인이라 부른다. 이는 비광 속 인물이 주인공의 그림자라는 사실에 대한 복선으로 사용한다.

12월 비광 속 인물 이벤트 방향:

- 비광 속 인물은 주인공이 방치한 세계를 물들인 주인공의 그림자다.
- 봉황의 깃털은 비광 속 인물을 정화하는 매개체다.
- 전투 초반의 비광 속 인물은 앞서 싸웠던 보스들의 특수 기술을 사용한다.
- 체력이 약 60%에 도달하면 주인공의 기술을 사용하기 시작한다.
- 체력 60% 도달 후 2~3턴이 지나면 주인공이 기시감을 느끼고, 비광 속 인물의 정체가 드러난다.
- 봉황의 깃털은 주인공이 기시감을 느낀 뒤, 보스 체력이 40%에 도달했을 때 사용할 수 있다.
- 혼돈치 낮음에서는 봉황의 깃털로 비광 속 인물을 정화하고, 그를 엑스트라 덱 세계의 관리자로 임명하는 방향을 검토한다.
- 혼돈치 중간에서는 봉황의 깃털 사용 대상을 선택한다. 본인에게 사용하면 그림자의 영향에서 벗어나 비광 속 인물을 죽이고 세계의 관리자가 된다. 비광 속 인물에게 사용해 성공하면 혼돈치 낮음과 같은 정화 엔딩으로 이어지고, 실패하면 혼돈치 높음과 같은 엔딩으로 이어진다.
- 혼돈치 높음에서는 주인공이 비광 속 인물의 힘을 흡수했기 때문에 힘의 관계가 역전되고, 역리가 순리가 되는 세계로 흐른다. 이후 사신수에 의해 저지당하는 방향을 검토한다.

전투 흐름:

```text
조우
→ 보스 체력 60% 도달
→ 기믹 전조 시작
→ 2~3턴 후 기믹 시작
→ 주인공이 기시감을 느낌
→ 보스 체력 40% 도달
→ 봉황의 깃털 선택지 발생
```

혼돈치별 전투 차이:

- 혼돈치 낮음: 디버프 없음. 공격 성공률 기준 80%.
- 혼돈치 중간: 디버프 1개. 공격 성공률 기준 60%.
- 혼돈치 높음: 디버프 2개. 공격 성공률 기준 30%.

디버프 후보:

- 특정 카드 필수 선택.
- 특정 카드 사용 불가 지정.

2회차 혼돈치 높음 엔딩:

- 혼돈치 높음 엔딩을 1차로 본 뒤 열리는 엔딩 분기로 검토한다.
- 혼돈치 중간에서 봉황의 깃털 사용에 실패해 혼돈치 높음과 같은 엔딩을 본 경우에도 열린다.
- 개방 상태는 로컬 저장값으로 관리한다.
- 상태값은 enum처럼 관리한다. `0`은 disable, `1`은 able이다.
- 혼돈치 높음 엔딩을 본 순간 값을 `1`로 바꿔 로컬에 저장한다.
- 프롤로그는 그대로 둔다. 프롤로그를 바꾸면 플레이어가 처음부터 혼돈치 높은 플레이를 해야 한다고 느낄 수 있다.
- 원래 선공권을 가지던 주인공이 2회차 혼돈치 높음 엔딩에서는 후공권을 가진다.
- 주인공은 반격과 공격을 이어가며 보스를 쓰러뜨린다.
- 보스를 쓰러뜨린 뒤 힘을 흡수하며 "이 또한 순리겠지"라고 말한다.
- 이후 인간 세계의 멸망을 보여주는 애니메이션이 나온다.
- 엔딩에서 할아버지와 아이가 있는 현실 프레임이 오염된다.
- 할아버지가 귀신 들린 것처럼 보스가 된 도사 이후 멸망한 세계를 이야기한다.
- 아이가 할아버지를 부르면 할아버지는 정신을 차리고, 자신이 원래 알던 이야기가 아니었다는 것을 깨닫는다.
- 마지막에 화투패 속 비광이 기괴한 웃음을 지으며 끝난다.

## 이벤트 타입 우선순위

MVP에서는 모든 이벤트 타입을 같은 비중으로 만들지 않는다. 1월~3월 범위에서 핵심 루프를 검증하는 데 필요한 이벤트부터 우선 제작한다.

우선순위:

1. 스토리 이벤트
2. 숨겨진 아이템 이벤트
3. 혼돈 이벤트
4. 보스 후속 이벤트
5. 파국 이벤트
6. 정비 이벤트
7. 동료 이벤트

MVP 포함:

- 스토리 이벤트
- 숨겨진 아이템 이벤트
- 혼돈 이벤트
- 보스 후속 이벤트

MVP 보류:

- 파국 이벤트
- 정비 이벤트
- 동료 이벤트

파국 이벤트는 혼돈치가 100까지 올라가는 장기 구조가 필요하므로 1월~3월 MVP에서는 보류한다. 정비 이벤트는 기본 정비 시스템이 먼저 안정된 뒤 추가한다. 동료 이벤트는 별도 동료 시스템의 범위가 커질 수 있으므로 후순위로 둔다.

## 이벤트 선택지 원칙

이벤트 선택지는 단순한 정답 찾기가 아니라 방향 선택이어야 한다.

- 기본 이벤트는 선택지 2개를 기준으로 한다.
- 중요한 이벤트는 선택지 3개까지 허용한다.
- 파국이나 강제 이벤트는 선택지 0~1개도 허용한다.
- 선택지 2개 이벤트는 지문을 제공한 뒤 한다/안 한다 구조로 사용할 수 있다.

- 안전한 선택: 손실이 적지만 보상도 낮다.
- 위험한 선택: 혼돈, 체력, 카드, 아이템을 대가로 큰 보상을 얻는다.
- 정보 선택: 즉시 보상은 낮지만 숨겨진 조건이나 보스 정보를 얻는다.
- 거절 선택: 이벤트를 지나치고 현재 상태를 보존한다.

선택지가 있는 이벤트는 최소한 아래 중 하나를 변화시켜야 한다.

- 혼돈치
- 체력
- 재화
- 카드 추가, 제거, 강화, 봉인
- 아이템 획득 또는 손실
- 동료 상태
- 숨겨진 플래그
- 이후 이벤트 또는 엔딩 조건

## 보상 위계

보상 가치는 아래 순서를 기준으로 설계한다.

```text
보스 보상
> 숨겨진 아이템 이벤트 보상
> 일반 전투 보상
> 일반 이벤트 보상
```

보스 보상은 월의 핵심 결산이며, 카드화, 소멸, 봉인, 특수 족보, 엑스트라 덱 카드, 큰 스토리 분기와 연결한다.

숨겨진 아이템 이벤트 보상은 일반 전투 보상보다 강하거나 고유해야 한다. 일반 전투 보상과 가치가 같으면 탐색 동기가 약해진다.

일반 전투 보상은 정비 재화를 중심으로 둔다.

```text
일반 전투
→ 정비 재화 획득
→ 전투 후 정비에서 성장 방향 선택
```

일반 전투 보상은 기본적으로 정비 재화이며, 낮은 확률로 카드 선택지를 추가 제공하는 방향을 검토한다. 이 경우 전투 보상 기대감은 생기지만, 덱 성장 루트가 복잡해질 수 있다.

일반 이벤트 보상은 일반 전투 보상보다 낮게 잡되, 정보, 리스크 회피, 작은 회복, 작은 혼돈 변화처럼 전투 보상과 다른 성격을 가진다.

## 혼돈치와 이벤트

혼돈치는 이벤트의 발생 조건과 결과를 바꿀 수 있다.

- 안정: 기본 이벤트가 주로 등장한다.
- 흔들림: 일부 이벤트 대사와 전투 전조가 변한다.
- 균열: 숨겨진 아이템 획득 조건에 가벼운 미니게임이나 추가 판정이 붙을 수 있다.
- 침식: 카드 봉인, 카드 삭제 압박, 특정 카드 사용 강제, 특정 카드 지정 같은 변칙이 이벤트 결과로 등장할 수 있다.
- 파국: 강제 이벤트가 발생해 게임이 바로 끝난다.

이벤트 왜곡은 아직 보류 중이다. 적용한다면 같은 이벤트라도 혼돈치에 따라 대사, 선택지, 결과가 바뀌는 방식으로 검토한다.

## 사흉수와 사신수 이벤트

사흉수 또는 사신수와의 만남은 혼돈치에 따라 발생하는 특수 이벤트 후보로 둔다.

기본 방향:

- 사흉수 이벤트: 높은 혼돈치, 불안정한 선택, 금기 사용과 연결한다.
- 사신수 이벤트: 낮거나 안정적인 혼돈치, 절제된 선택, 철학적 판단과 연결한다.

사흉수의 시험:

- 전투 중심 시험.
- 환각, 모순 찾기, 왜곡된 선택지 같은 시험을 포함할 수 있다.
- 플레이어가 힘을 대가로 무엇을 포기했는지 묻는 방향이 어울린다.
- 성공 시 강한 보상이나 혼돈 빌드 지원을 제공할 수 있다.
- 실패 시 혼돈치 증가, 카드 봉인, 체력 손실, 이벤트 플래그 왜곡 같은 리스크를 줄 수 있다.

사신수의 시험:

- 퀴즈, 문답, 철학적 선택 중심 시험.
- 세계관, 월의 상징, 플레이어의 이전 선택을 되묻는 구조가 어울린다.
- 정답 하나를 맞히는 방식보다, 플레이어가 어떤 가치를 택하는지 확인하는 방향을 우선한다.
- 성공 시 혼돈 안정, 정보, 방어적 보상, 엔딩 조건 보정 같은 효과를 줄 수 있다.
- 실패해도 큰 처벌보다 기회 상실이나 작은 리스크를 주는 방향을 검토한다.

이 이벤트들은 초기 MVP 필수 요소는 아니다. 혼돈치 시스템과 시나리오 구조가 더 구체화된 뒤, 월별 특수 이벤트 또는 중후반 이벤트로 배치한다.

## 숨겨진 아이템 이벤트

모든 월에는 숨겨진 아이템을 배치한다.

숨겨진 아이템은 발견 조건을 만족하면 기본적으로 획득 확정으로 둔다. 선택지는 획득 여부를 고르는 용도보다, 발견 장면을 보여주거나 혼돈치에 따른 추가 판정을 처리하는 용도로 사용한다.

숨겨진 아이템 보상 원칙:

- 고유 효과를 가진다.
- 특정 빌드를 지원한다.
- 일부는 혼돈치를 증가시킬 수 있다.
- 발견 조건이나 미니게임이 필요할 수 있다.
- 월별 1개 제한을 기본으로 한다.

숨겨진 아이템은 단순한 보물상자가 아니라, 아래 조건 중 하나 이상을 요구하는 방식으로 설계한다.

- 특정 월의 전투를 특정 방식으로 클리어
- 특정 카드 타입 보유
- 특정 보스 처리 선택 이력
- 특정 혼돈치 구간
- 특정 정비 선택
- 가벼운 미니게임 성공
- 이전 월에서 얻은 단서 보유

숨겨진 아이템 중 일부는 강력한 대신 혼돈치를 증가시킬 수 있다.

혼돈치에 따른 숨겨진 아이템 처리:

- 낮은 혼돈치: 조건을 만족하면 바로 획득한다.
- 높은 혼돈치: 조건을 만족해도 가벼운 미니게임이나 추가 판정이 붙는다.
- 미니게임 실패 시 아이템을 잃게 할지, 아이템은 얻되 추가 리스크를 받을지는 추후 결정한다.

숨겨진 아이템 단서는 전투 보상으로 직접 제공하지 않는다. 숨겨진 아이템 근처에 도달했을 때 캐릭터 대사로 넌지시 암시하는 방식을 우선 검토한다.

숨겨진 아이템이 매 월 존재한다는 사실은 플레이어에게 명시하지 않는다. 우연히 발견했을 때의 재미를 우선한다.

대신 아래 방식으로 존재 가능성을 은근히 암시한다.

- 캐릭터 대사
- 월별 배경 묘사
- 특정 장소의 어색한 연출
- 반복 플레이 중 알아차릴 수 있는 작은 패턴

혼돈 관련 기본 정보는 단서 보상보다 팁박스나 기본 도움말로 제공한다. 구체적인 활용과 위험은 플레이를 통해 알아가게 한다.

## 파국 이벤트

혼돈치가 100에 도달하면 파국 이벤트가 발생한다.

파국 이벤트는 보스를 다른 존재로 바꾸는 방식보다, 현재 보스와 플레이어의 관계를 강제로 결론짓는 방식으로 설계한다.

후보:

- 보스의 아래로 들어간다.
- 보스와 즉시 결투하고, 패배 또는 승리로 게임이 끝난다.
- 승리 후 플레이어가 새로운 보스가 된다.

새로운 보스가 되는 엔딩은 2회차 이후 개방하는 방향을 검토한다.

## JSON 데이터화 방향

이벤트는 추후 JSON 파일로 옮겨 관리한다.

JSON 스키마는 구현 전까지 1차 안정 스키마로 관리한다. 공통 필드는 먼저 고정하되, `conditions`와 `effects` 내부 세부 필드는 카드, 혼돈, 보스, 시나리오 기획이 더 구체화된 뒤 확정한다.

`id`와 `type`은 DB 테이블 개념이 아니라 게임 데이터 식별과 분류를 위한 필드다.

- `id`: 이벤트 하나를 고유하게 식별하는 값.
- `type`: 이벤트 처리와 필터링을 위한 분류값.

이벤트 파일은 월 또는 범주별로 나누고, 파일 안에서는 여러 이벤트를 배열로 관리한다.

예상 구조:

```text
Data/
  Events/
    month_01.json
    month_02.json
    month_03.json
    common.json
    chaos.json
    boss_aftermath.json
```

파일 기본 구조:

```json
{
  "events": []
}
```

선택지가 없는 이벤트도 같은 구조로 관리한다. 이 경우 `choices`는 생략하지 않고 빈 배열로 둔다.

1차로 고정할 공통 필드:

- `id`
- `type`
- `title`
- `month`
- `priority`
- `once`
- `conditions`
- `script`
- `choices`
- `effects`

아직 열어둘 필드:

- `conditions` 내부 세부 조건
- `effects` 내부 세부 효과
- 미니게임 관련 필드
- 동료 이벤트 전용 필드
- 파국 이벤트 전용 필드

`conditions`와 `effects`는 시나리오 작성 이후 실제 필요한 조건과 결과를 기준으로 구체화한다.

초기 JSON 필드:

```json
{
  "events": [
    {
      "id": "event_month01_hidden_flower_01",
      "type": "hidden_item",
      "title": "숨겨진 꽃패",
      "month": 1,
      "priority": 10,
      "once": true,
      "conditions": {
        "chaos_min": 0,
        "chaos_max": 49,
        "required_flags": [],
        "required_items": [],
        "required_cards": []
      },
      "script": {
        "default": "낡은 장독대 뒤에서 희미한 기운이 느껴진다.",
        "hard": "장독대 뒤의 기운은 처음부터 너를 기다렸다는 듯 흔들린다."
      },
      "choices": [
        {
          "id": "inspect",
          "label": "살펴본다",
          "requirements": {},
          "effects": {
            "gain_item": "hidden_flower_january",
            "set_flags": ["found_january_hidden_item"]
          }
        },
        {
          "id": "leave",
          "label": "그냥 지나친다",
          "requirements": {},
          "effects": {}
        }
      ],
      "effects": {}
    }
  ]
}
```

선택지가 없는 이벤트 예시:

```json
{
  "id": "event_month01_intro_01",
  "type": "story",
  "title": "첫 달의 문턱",
  "month": 1,
  "priority": 100,
  "once": true,
  "conditions": {},
  "script": {
    "default": "첫 번째 달의 문이 열린다."
  },
  "choices": [],
  "effects": {
    "set_flags": ["seen_month01_intro"]
  }
}
```

이벤트 타입은 문자열 enum처럼 관리한다. 초기 타입은 `story`, `hidden_item`, `chaos`, `boss_aftermath`, `maintenance`, `catastrophe`, `companion`으로 둔다.

## 미정 사항

- 이벤트 왜곡을 실제 시스템으로 넣을지 여부
- 사흉수/사신수 이벤트를 실제 시스템으로 넣을지 여부
- 사흉수/사신수 이벤트의 발생 조건과 보상
- 숨겨진 아이템 미니게임의 구체 방식
- 동료 이벤트를 초기 버전에 넣을지 여부
- `conditions`와 `effects` 내부 세부 필드
